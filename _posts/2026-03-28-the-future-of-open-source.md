title: "The Future Of Open Source"
---

Open source software has always operated on a kind of implicit social contract. Someone writes something useful, puts it in public, and other people build on it, fix it, and improve it. The maintainer gets contributions and visibility. The users get software they didn't have to write themselves. It's not a perfect system, but the upsides have been importantly useful in the internet age.

That contract is now under new pressure.

The most immediate tension is with AI models trained on this open source code. The code is public. The licenses, in most cases, permits it. But the relationship between contributing to a commons and having that commons used to train a commercial product that competes with the people who built it sits uncomfortably with a lot of maintainers. The contribution is made in one spirit, yet used in another.

Some projects have responded by changing their licenses. For example, HashiCorp moved Terraform to a Business Source License. Redis followed. The argument is the same. Loud providers were profiting from the work without giving back. AI has sharpened that argument further. The open source definition hasn't changed, but what it means to make your work public has.

At the same time, AI is genuinely accelerating open source contribution. More people can write code now, which means more people can submit pull requests, fix bugs, and start projects. In turn, the volume of new repositories, new tools, and new libraries has increased. Whether that volume translates to quality, maintained software or a flood of abandoned prototypes is still playing out.

The more interesting question is what open source looks like when the primary consumer of code isn't a developer reading it but a model ingesting it. Documentation written for humans serves a different purpose than documentation that helps a model understand intent. Licensing written before AI existed doesn't cleanly map onto a world where your code can be pattern-matched and reproduced without being directly copied.

What's likely is that open source doesn't go away; it's too foundational, and the incentives to share haven't disappeared. What probably changes is the layer of formality around it. More projects thinking carefully about licensing from the start. More distinction between code that's open to read and code that's open to build on and code that's open to train on. The lines that were blurry because nobody needed them to be clear are going to need to be clearer.

The social contract isn't broken. It's just being renegotiated.
